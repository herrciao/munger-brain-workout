# 英文版雙語切換方案（修訂版）— 回溯原文策略

## 核心洞察：不是「翻譯」，是「回溯原文」

用戶的 12 本參考書幾乎全是英文原版。現有題目中的理論名稱、作者名、書名本來就是英文。因此策略從「翻譯 133,400 字中文」變成：

- **直接提取**：模型英文名（已在括號內）、理論名稱、作者名、書名
- **參照原文重寫**：解析和錯誤原因中的理論描述，直接用英文原文書的術語和語境
- **英文創作**：商業情境（scenario）用英文重新描述（非逐字翻譯）
- **查找原句**：芒格語錄從英文世界找 Poor Charlie's Almanack 原句

## 各欄位的工作性質

| 欄位 | 工作性質 | 說明 |
|------|---------|------|
| models / secondaryModels | 直接提取 | 括號內已有英文名如 `Inversion`、`Anchoring`，幾乎零工作量 |
| explanation / wrongReasons | 半提取半重寫 | 理論名、書名、作者名直接用英文；論述邏輯用英文重寫 |
| scenario | 英文重寫 | 原創商業情境，無原文可回溯，用英文重新描述 |
| options | 英文重寫 | 概念名用英文原文，描述部分重寫 |
| UI 文字 | 簡單翻譯 | 按鈕、標題等短句 |
| 芒格語錄（19 條） | 查找原句 | 從 Poor Charlie's Almanack 英文版找原句 |

## 技術架構

沿用純 HTML/JS 輕量架構，不用任何框架：

- 題目格式新增 `en_` 前綴欄位（`en_scenario`, `en_options`, `en_explanation`, `en_wrongReasons`, `en_models`）
- 新增 `i18n/` 目錄放 UI 文字和語錄的語言包
- `index.html` 加入語言切換按鈕 + `lang` 全域變數 + `t()` helper

## 品質優勢

回溯原文策略的好處不只是速度，更重要的是**品質**：

- 用 Kahneman 原文的 "prospect theory"、"loss aversion" 比翻回去更精確
- 用 Cialdini 原文的 "commitment and consistency" 比「承諾與一致性」更道地
- 用 Taleb 原文的 "antifragile"、"skin in the game" 是原汁原味
- 英文讀者會看到他們熟悉的術語，而非翻譯腔
